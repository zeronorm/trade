一级目录规划
src
    core/ (核心逻辑层)
    宏观作用：系统的“中央处理器”。负责主引擎的调度，维护账户的资金流转、实时持仓状态和订单生命周期，负责把其他所有独立模块串联起来运作。
    
    data/ (市场数据层)
    功能：支持天级全量股票更新（crontab)，按天级保存数据然后合并；注意股票数目1万左右，2年数据
    支持股票：a股+港股+美股 历史日K线和每日最新横切面快照

    细节:
    - 使用akshare基础库, 
    - 实时行情(所有上市公司, 以{a/us/hk}.{date}.csv 保存; 每天在对应市场收盘后定时更新
      - 中:stock_sh_a_spot(_em) stock_sz_a_spot(_em) stock_bj_a_spot(_em) stock_cy_a_spot(_em) stock_kc_a_spot(_em)
      - 港美:stock_zh_a_daily同理
    - 历史行情stock_zh_a_daily(股票数目多，单词完成需要很久时间）
      - 以{symbol}.{end_data}.history_2y.csv 保存
    - 如果数据存在则跳过不用再次获取
    - 如果有新股, 需要触发一次历史行情数据的抓取

    strategy/ (交易策略层)
    宏观作用：赚钱的“业务逻辑”。存放各种量化模型和交易算法，只负责接收行情数据并生成相应的买卖交易信号，不关心底层如何发单。
    
    risk/ (风控安全层)
    宏观作用：防爆仓的“防火墙”。在主引擎向交易所发送任何订单前，强制进行事前拦截与校验（如资金校验、乌龙指防范、最大回撤限制）。
    
    backtest/ (历史回测层)
    宏观作用：策略的“实验室”。复用 strategy 的代码，提供一个模拟的撮合引擎和绩效分析工具，验证策略在历史数据下的表现。
    
    utils/ (通用工具层)
    宏观作用：底层“基础设施”。提供所有模块都会用到的通用公共组件，如日志记录器、全局配置解析、自定义异常类和时间格式转换工具。
    
    config/ (配置文件层)
    宏观作用：系统的“控制面板”。集中管理不同环境（开发、回测、实盘）的运行参数、敏感的 API 密钥以及数据库连接字符串，与代码逻辑严格分离。


notebooks/ (交互研究层)

宏观作用：量化研究的“草稿本”。存放 Jupyter Notebook (.ipynb) 文件。主要用于因子挖掘、数据清洗探索、策略逻辑的交互式预研，以及回测结果的深度图表可视化分析。

examples/ (调用演示层)

宏观作用：快速上手的“说明书”。存放一系列独立的 .py 脚本，演示如何组装和调用 src 中的各个模块。比如：run_backtest_demo.py（演示回测）、connect_binance_demo.py（演示行情接入）、simple_grid_bot.py（一个完整的简单网格交易机器人示例）。

scripts/ (辅助运维层)

宏观作用：日常维护的“工具箱”。存放脱离主交易进程运行的独立脚本和定时任务（Cron Jobs）。例如：周末定时下载历史 K线数据的脚本、数据库初始化建表脚本、持仓对账和报警脚本。

tests/ (自动化测试层)

宏观作用：系统安全的“质检员”。存放使用 pytest 或 unittest 编写的测试用例。在金融交易系统中至关重要，特别是针对资金计算、风控拦截规则和撮合逻辑的单元测试，防止代码迭代导致实盘亏损。

config/ (配置管理层)

宏观作用：系统的“控制面板”。存放不同环境（开发、回测、实盘）的 .yaml 或 .json 配置文件、API Keys 等（通常配合 .gitignore 保护敏感信息），与代码逻辑严格物理隔离。

data_store/ (本地数据层)

宏观作用：静态资产的“仓库”。存放非代码类型的文件。例如下载好的历史 CSV/Parquet 行情数据、本地 SQLite 数据库文件、以及回测跑完后输出的 HTML/Excel 绩效报告。
